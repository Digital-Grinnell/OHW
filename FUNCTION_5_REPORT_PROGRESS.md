# Function 5: Report Workflow Progress

## Purpose
Generate a comprehensive progress report comparing files in the input directory with processed files in ~/OHM-data. This helps track which interviews have been processed and identify what work remains.

## Requirements
- **Input directory selected**: Must have selected a directory containing audio files
- **No prerequisites**: Can be run at any time to check status

## Usage

1. Make sure you have selected an **input directory** containing your audio files
2. In the **Active Functions** dropdown, select **"📊 5: Report Workflow Progress"**
3. The function will:
   - Scan your input directory for WAV and MP3 files
   - Scan ~/OHM-data for processed files and directories
   - Compare what exists vs. what's been processed
   - Generate a timestamped markdown report
4. Monitor the status and log output for results

## Report Location

The report is saved in:
```
~/OHM-data/workflow_progress_YYYYMMDD_HHMMSS.md
```

Example:
```
~/OHM-data/workflow_progress_20260407_143022.md
```

Each report is timestamped, so you can keep historical snapshots of your progress.

## Report Contents

The generated report includes:

### Overview Section
- Report generation timestamp
- Input and output directory paths
- Workflow stages explanation

### Summary Statistics
- Total files in input directory
- Total processed directories
- Complete files count
- In-progress files count
- Not started files count

### Processing Stages Table
- MP3 files completed
- JSON transcripts created
- TXT outputs generated
- VTT outputs generated
- CSV outputs generated
- PDF outputs generated

### Detailed File Status

Files are categorized into three sections:

#### ✅ Complete Files
- All stages finished (MP3, JSON, TXT, VTT, PDF)
- Shows directory location
- Ready for archival/distribution

#### 🟡 In Progress Files
- Some stages completed
- Lists missing components
- Shows directory location
- Helps identify what needs to be finished

#### ⏳ Not Started Files
- Only source file exists in input directory
- Haven't been processed yet
- Shows next recommended step

### Workflow Reminder
- Step-by-step processing instructions
- Clarifies the complete workflow from audio to final outputs

## Use Cases

### Track Overall Progress
Run periodically to see how many interviews have been processed and how many remain.

### Identify Incomplete Work
Quickly find which files are partially processed and what's missing.

### Quality Assurance
Verify all files have gone through the complete workflow before final delivery.

### Documentation
Keep timestamped snapshots of progress for project records.

## Workflow Stages Tracked

The report tracks files through these stages:

1. **Audio (WAV/MP3)** - Original or converted audio file
2. **Transcription (JSON)** - Created by Function 2 (Whisper or MS Word Online)
3. **Outputs (TXT, VTT, CSV, PDF)** - Generated by Function 4

## Notes

- **No side effects**: This function only reads data and generates a report; it doesn't modify any files
- **Run anytime**: Safe to run as often as you like to check progress
- **Historical tracking**: Each report is timestamped, allowing you to track progress over time
- **Self-documenting**: Reports include the workflow reminder to help team members understand the process

## Next Steps After Reviewing Report

Based on what the report shows:

- **Files not started**: Run Function 1 (if WAV) or Function 2 (if MP3)
- **Missing transcripts**: Run Function 2 to create JSON transcripts
- **Missing outputs**: Run Function 4 to generate TXT, VTT, CSV, and PDF files
- **Incomplete files**: Check the "In Progress" section to see what's missing

---

**Tip:** Keep the latest report open while working to track your progress!
